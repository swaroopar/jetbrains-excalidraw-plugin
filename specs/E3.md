Edits are written back to the opened file automatically after a debounce window,
with no explicit save action required. Persistence goes through the IDE Document /
Virtual File System so that edits integrate with IDE save, undo, and
external-change detection.

The serialized output conforms to the canonical `.excalidraw` JSON schema
(`type`, `version`, `source`, `elements`, `appState`, `files`) so files remain
interoperable with excalidraw.com and the VSCode plugin.
