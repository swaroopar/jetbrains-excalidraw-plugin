# E1 — Open a `.excalidraw` file in a custom editor

When the user opens a `.excalidraw` file in the IDE, the plugin's custom editor
(`FileEditorProvider` / `FileEditor`) is used instead of the plain text editor. It
hosts the Excalidraw web app in a JCEF browser, reads the file via the Virtual
File System, and renders its scene (elements + app state) on the canvas.

A corrupt or schema-invalid file must surface a non-destructive error and must
not overwrite the source file or crash the editor.

Scene-embedded `.excalidraw.png` files are also handled by this editor; their
scene extraction and re-embedding are specified separately in E6.

## Acceptance Criteria

- AC-E1-01: Opening a valid `.excalidraw` file opens the Excalidraw custom editor
  and renders the canvas with all elements and app state from the file.
- AC-E1-02: Opening an invalid or corrupt file shows an error message, leaves the
  source file untouched, and does not crash the editor.
