# E6 — Treat `.excalidraw.png` as an editable Excalidraw file

Files with the `.excalidraw.png` extension are scene-embedded PNGs: a valid PNG
image that also carries the Excalidraw scene in its metadata. The plugin must
treat them as first-class Excalidraw files in the custom editor, exactly like the
VSCode Excalidraw plugin:

- On open, detect `.excalidraw.png`, extract the embedded scene, and render it on
  the canvas (using Excalidraw's blob-loading utilities).
- Edit behaves identically to a plain `.excalidraw` file (E2).
- On auto-save (E3), re-embed the updated scene into the PNG and write it back via
  the VFS, so the file stays both a valid PNG image and a re-openable Excalidraw
  file.

A `.excalidraw.png` that carries no embedded scene data is handled like any other
invalid source (non-destructive error, source untouched — see AC-E1-02).

## Acceptance Criteria

- AC-E6-01: Opening a `.excalidraw.png` with an embedded scene extracts that scene
  and renders all its elements and app state on the canvas.
- AC-E6-02: Auto-saving a `.excalidraw.png` re-embeds the current scene into the
  PNG; the written file is both a valid PNG and re-openable as an Excalidraw file
  with the same scene.
- AC-E6-03: Opening a `.excalidraw.png` without embedded scene data shows a
  non-destructive error and leaves the source file untouched.
