# E7 — Configurable file extensions (IDE settings)

The plugin provides a settings panel inside the IDE (registered as a
`com.intellij.openapi.options.Configurable`, exposed under
**Settings / Preferences → Tools → Excalidraw**) where the user can view and
edit the list of file extensions and filename patterns that are treated as
Excalidraw files and opened in the custom editor.

The default list is `.excalidraw` and `.excalidraw.png`, matching the
hard-coded behaviour of earlier iterations and the VSCode Excalidraw plugin
defaults.

The configuration is persisted via a **`PersistentStateComponent`** at
Application level (IntelliJ Platform persistence API), so the list survives IDE
restarts and is shared across projects opened in the same IDE installation.

`ExcalidrawFileEditorProvider.accept()` consults the configured extension list
instead of hard-coded suffixes when deciding whether to open a file in the
custom editor. Changes to the list take effect for newly opened files; files
already open in the editor are not affected.

This feature mirrors the configurable file-association support of the VSCode
Excalidraw plugin, giving power users the ability to add custom extensions
(e.g. `.excd`) or to remove default ones.

## Acceptance Criteria

- AC-E7-01: The IDE Settings dialog contains an Excalidraw panel (Tools >
  Excalidraw) that displays the current list of file extensions and allows the
  user to add, edit, and remove entries.
- AC-E7-02: On a fresh installation (or after resetting to defaults) the
  configured extension list contains exactly `.excalidraw` and
  `.excalidraw.png`.
- AC-E7-03: Adding a new extension (e.g. `.excd`) causes
  `ExcalidrawFileEditorProvider.accept()` to return `true` for files with that
  extension; removing `.excalidraw` from the list causes `accept()` to return
  `false` for plain `.excalidraw` files.
- AC-E7-04: The configured extension list is preserved across IDE restarts
  (i.e. stored via `PersistentStateComponent` and reloaded on startup).
