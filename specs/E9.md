# E9 — Robust editor loading & rendering (JCEF integration contract)

The custom editor must actually open and render the Excalidraw web app inside the
IDE's JCEF browser for every accepted file. This requirement captures integration
behaviours that are NOT visible in unit tests (they only manifest when a real IDE
loads the plugin) and that regressed the editor to "not opening" or "blank window"
during the first implementation. They are mandatory and must not be lost on a
re-build.

## Behaviour

1. **Empty / newly-created files open as a blank canvas.** Opening a 0-byte or
   whitespace-only `.excalidraw` (e.g. created via *New File*) opens a usable blank
   Excalidraw canvas, NOT a parse-error notification. Genuinely malformed (non-empty)
   JSON still surfaces a parse error and does not overwrite the file.
2. **`.excalidraw.png` opens in the Excalidraw editor, not the image viewer.** The
   file-type association uses a filename **pattern** (`*.excalidraw.png`), because the
   IntelliJ platform derives a file's extension from the last dot only (`png`), so an
   `extensions="excalidraw.png"` association never matches and the file would open as
   a generic PNG image.
3. **The bundled web app renders.** index.html and all of its sub-resources
   (`bundle.js`, `bundle.css`, fonts) load and the Excalidraw canvas + toolbar appear.

## Acceptance Criteria

- AC-E9-01: Opening an empty or whitespace-only `.excalidraw` file shows a blank
  Excalidraw canvas (toolbar visible), with no parse-error notification and no write
  to the file until the user edits.
- AC-E9-02: Opening a file with malformed (non-empty) JSON shows a parse-error
  notification and does not open a corrupt/blank scene as if valid.
- AC-E9-03: A `.excalidraw.png` file opens in the Excalidraw custom editor, not the
  IDE image viewer (the file type is matched via the `*.excalidraw.png` pattern).
- AC-E9-04: When the editor opens, the JCEF browser successfully loads
  `excalidraw://app/index.html` AND its sub-resources (`bundle.js`, `bundle.css`),
  i.e. the served content is treated as HTML and the scripts/styles are not refused.
- AC-E9-05: `.excalidraw` and `.excalidraw.png` files display the Excalidraw icon
  (see E8).
